Tema A -> A1 — Playlist de Músicas

Tema B -> B2 — Chamadas de Emergência

Tema C -> C1 — Atendimento Bancário

Resumo do programa A
A ideia é criar um sistema simples de playlist, em que cada música cadastrada será armazenada em um vetor de struct.

O programa deve permitir cadastrar músicas, buscar, editar, excluir e listar os registros.

Resumo do programa B
A ideia é simular uma pilha de chamadas de emergência.

A última chamada registrada será a primeira chamada atendida, respeitando a lógica LIFO.

Resumo do programa C
A ideia é simular um sistema de atendimento bancário com senhas normais e preferenciais.

O programa deverá manter duas filas:

uma fila normal simples;
uma fila circular simples para senhas preferenciais.

Instruções para compilar e executar
Copie o conteudo dos arquivos .c e execute na sua IDE de preferencia, atente-se para pasta ou modulos da sua IDE para que não tenha conflito para gerar ou carregar os arquivos CSV.

Observações
- Alguns testes feito no CLion não executou o comando system("clear"), devido a configuração do terminal da propria IDE

- No trabalho C tem alguns comando system("clear"), o clear funciona no linux mas não no windows, caso queria a tela limpando, selecione com ctrl+f e substitua por system("cls")

- Atente-se de colocar os arquivos CSV na mesma pasta do projeto caso escolha rodar no onlinegdb
